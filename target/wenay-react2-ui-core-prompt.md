# Промпт для репозитория wenay-react2: три проектно-нейтральных UI-механизма

Скопируй текст ниже в сессию в репе wenay-react2. Механизмы нужно реализовать в
библиотеке в обобщённом виде. Библиотека НИЧЕГО не знает о
содержимом: разделы, контент слотов и колбэки приходят снаружи (как у ButtonOutClick,
DivRnd3). Стиль оформления — как у окон (--wnd-*): CSS-переменные с дефолтами.

---

Нужно добавить три механизма. Ниже контракты; детали реализации на твоё усмотрение,
но публичные API сохранить как описано — приложения переводятся на них механически.

## 1. Диалог настроек по центру + реестр разделов

Поведение (эталон — SettingsMenu в приложении):
- кнопка-триггер (передаётся снаружи) открывает диалог по центру экрана:
  скрим на весь вьюпорт (клик по скриму закрывает), панель ~640×420 (max 92vw/82vh),
  слева колонка разделов, справа контент активного раздела, крестик в правом углу контента;
- разделы двух видов: статические (передаются пропом) и ВНЕШНИЕ — любой модуль
  регистрирует раздел в реестре на маунте и снимает на анмаунте.

API:
```ts
export type tSettingsSection = {key: string, name: string, render: () => React.ReactNode}

// реестр: множественная регистрация, повтор по key заменяет, возврат = снятие
export function registerSettingsSection(s: tSettingsSection): () => void

export function SettingsDialog(props: {
    trigger: React.ReactNode              // что рендерить как кнопку (обёртка кликабельна)
    sections?: tSettingsSection[]         // статические разделы (первые в списке)
    defaultSection?: string
}): JSX.Element
```
Реестр — модульный синглтон (замыкание + подписка, БЕЗ React-контекста), диалог
подписан на его изменения. Клик по разделу слева — контент справа; активный раздел
подсвечен.

Стили — через CSS-переменные с дефолтами (тёмные, близкие к текущим приложениям),
по образцу --wnd-*: `--dlg-scrim`, `--dlg-bg`, `--dlg-border`, `--dlg-radius`,
`--dlg-shadow`, `--dlg-nav-bg`, `--dlg-nav-width`. Кнопки разделов — классы
приложения не использовать; либо свои минимальные стили, либо принять
`sectionClassName / sectionActiveClassName` пропами (лучше пропы — приложения
дадут свои .chip/.chipActive).

## 2. Слот с настраиваемым местом размещения

Поведение (эталон — strategyBar.tsx): один блок UI может показываться в одном из
нескольких мест; место — настройка, персистится; точки монтажа сами решают,
рендерить ли им контент.

API:
```ts
export function createUiSlot<Places extends string>(opts: {
    key: string                            // ключ персиста (staticGetAdd/Cash)
    places: {[P in Places]: string}        // место → человеческое имя
    def: Places
}): {
    Slot: (p: {place: Places, children: React.ReactNode}) => JSX.Element | null
    PlacementSetting: (p?: {className?: string, activeClassName?: string}) => JSX.Element
    getPlace: () => Places
    setPlace: (p: Places) => void
}
```
Персист и реактивность — существующей механикой библиотеки (staticGetAdd +
updateBy/renderBy + Cash). PlacementSetting — ряд кнопок-сегментов по places.

## 3. Мультиплексор одиночного колбэка

Проблема: многие API отдают ОДИН слот-колбэк (`onX(cb | null)`), подписчики
перезаписывают друг друга молча (реальный пойманный баг).

API:
```ts
export function createCallbackHub<Args extends any[]>(
    bind: (emit: (...a: Args) => void) => void   // вызовется лениво, один раз, при первом on()
): {
    on: (cb: (...a: Args) => void) => () => void  // подписка; возврат = отписка
    count: () => number
}
```
Никакого UI. bind вызывается при первом подписчике (не в момент создания —
слот может быть ещё занят инициализацией приложения).

## Проверка

- сборка + типы; демо: диалог с двумя статическими разделами + один
  зарегистрированный извне (появляется/исчезает с маунтом компонента);
- слот с двумя местами — переключение в PlacementSetting перемещает контент без
  перезагрузки, выбор переживает reload (Cash);
- hub: две подписки на один слот получают события обе; off одной не ломает вторую.

## Не делать

- Никакой доменной логики/имён из приложений (страницы, темы, балансы — снаружи).
- Не менять существующие API (ButtonOutClick, DivRnd3, меню).
