# Промпт для репозитория wenay-react2: темизация окон DivRnd3 под макет приложения

Скопируй текст ниже в сессию в репе wenay-react2.

---

Нужно привести «хром» плавающих окон DivRnd3 (рамка, шапка-drag-зона, крестик закрытия)
к виду согласованного макета приложения и сделать его темизируемым через CSS-переменные.
Сейчас стили окна жёсткие и не реагируют на смену темы приложения.

## Контекст

Приложения-потребители (clientBacktest / clientAdmin) переключают темы атрибутом
`[data-theme]` на `<html>` и перекрывают токены библиотеки селектором
`:root[data-theme='...']` (он выигрывает у библиотечного `:root` по специфичности).
Так уже темизированы `--color-*` и `--menu-*`. Для окон нужен такой же контракт.

## Задача

1. Ввести переменные окна (объявить дефолты в инжекте токенов библиотеки +
   зеркало в `src/common/src/styles/tokens.ts`). Дефолты = ТЕКУЩИЙ вид, чтобы
   не изменить приложения без тем:
   - `--wnd-bg` — фон окна (сейчас тёмная панель);
   - `--wnd-border` — рамка (например `1px solid <текущий цвет>`);
   - `--wnd-radius` — скругление (в макете 10px, дефолт — текущий);
   - `--wnd-shadow` — тень (в макете: `0 2px 6px rgba(0,0,0,.45), 0 14px 40px rgba(0,0,0,.35)`);
   - `--wnd-header-bg` — фон шапки-drag-зоны;
   - `--wnd-header-color` — цвет заголовка;
   - `--wnd-header-border` — нижняя рамка шапки;
   - `--wnd-close-color` / `--wnd-close-hover` — крестик.

2. Перевести стили DivRnd3 на эти переменные. Целевой вид по макету
   (приложение задаст значения токенами, библиотеке важна СТРУКТУРА):
   - окно: фон `--wnd-bg`, рамка `--wnd-border`, радиус `--wnd-radius`, тень `--wnd-shadow`;
   - шапка: высота ~28px, padding `6px 10px`, фон `--wnd-header-bg`,
     нижняя рамка `--wnd-header-border`; заголовок 11–12px, letter-spacing .5px,
     цвет `--wnd-header-color`, без text-transform (не капсить);
   - крестик: справа в шапке, кликабельная зона ≥ 24×24, цвет `--wnd-close-color`,
     hover → `--wnd-close-hover`, без рамок/фона;
   - контент окна: без изменений (padding не трогать — им управляют приложения).

3. Не менять: механику drag/resize, ExRNDMap3-персистенцию, MemoChild/замораживание
   children, публичные пропсы. Только визуальный слой.

4. Проверка: собрать, открыть демо/любое окно — вид 1:1 с текущим при дефолтах;
   затем в devtools задать на `:root` другие `--wnd-*` — окно перекрашивается без перезагрузки.

## Примечание

Тем же заходом, если просто: у меню (menuRight.css) переменные `--menu-*` уже есть —
убедиться, что ВСЕ цвета меню идут через них (без остаточных литералов).
