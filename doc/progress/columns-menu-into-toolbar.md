# Кнопки меню столбцов ВНУТРЬ функционала «Макета 25» (createToolbar)

**Goal:** кнопки нашего меню столбцов (показать/скрыть столбец) должны жить внутри функционала тулбара «card 25» (`createToolbar`) — тот же каркас: тёмная рамка, шестерёнка и настройка «какие элементы меню видны».

**Эталон НЕ трогаем:** card 25 = `ToolbarDemo` / `qaToolbar` в `qa.tsx` (~907). Только читаем как образец вида.

Разбито пользователем на **2 этапа**:
- **Этап 1 (этот):** внести кнопки в тулбар. Демо-уровень (`qa.tsx`), библиотечные компоненты не трогаем.
- **Этап 2 (потом):** расширить API тулбара + прокинуть колбэк — перестановка столбцов в гриде зовёт API меню (двусторонняя связь порядка).

## Обратные колбэки / пересчёт прайса → «идентичные кнопки» — РЕШЕНО (через хук-api, отдельным слоем)
- Запрос: при пересчёте прайса таблица сама меняет колонки → кнопки должны оставаться идентичны таблице; предпочтение — **через api, которое возвращает хук, отдельным слоем**.
- **Presence/видимость (это и есть «пересчёт»):** реверс уже реактивный через **api-хук** — `columnState.api.usePresent()` (грид-адаптер зовёт `setPresent` на событиях грида → `usePresent` перерисовывает потребителей). `MenuStateButton` его и читает → на смене стандарта колонки уходят из грида, кнопки становятся `disabled` (пунктир) и оживают обратно, оставаясь идентичны таблице. Отдельный реактивный слой, без ручного бриджа. Это ровно предпочтённый механизм.
- **Порядок:** теперь не write-write bridge, а order-only source внутри `createToolbar`; внешний `columnState.api.listSource` отдаёт порядок, toolbar сохраняет локальные extra-slots и membership.
- Check card 30 (`do/expect/note`) обновлён под `sourceMode:"order"` + под FloatingWindow-настройки (старые упоминания FLIP/твитча убраны — FLIP был откачен).
- `tsc -p tsconfig.qa-check.json` зелёный; добавлен unit-тест `createToolbar.test.ts` на order-only source.

## Ключевые решения
- **`sourceMode:"order"` на этом тулбаре.** Обычный `source` (`orderVisible`) связал бы «видимость в баре» с «видимостью в гриде», и ненажатые кнопки исчезали бы из бара. Новый режим берёт из `columnState` только порядок source-ключей.
  - Слой 1 (членство): локальный `visible` тулбара — управляется шестерёнкой/`Settings`.
  - Слой 2 (нажат/ненажат): `qaMenuState.visible[col]` — видимость столбца в гриде, её показывает и переключает сама кнопка.
  - Слой 3 (порядок): `columnState.api.listSource.order` для реальных колонок; extra item `blockMode` остаётся локальным и не пишется в grid config.
- Стилизация кнопок под тёмную рамку — **временная демо-стилизация**; на этапе 2 причешем/переиспользуем.
- **Порядок бара синхронизирован с гридом** через order-only source; grid меняет только порядок меню, не трогая членство/плотность.
- Changelog обновляем: focused API, `sourceMode:"order"` и reset-псевдокнопка — публичное расширение `createToolbar`; demo card 30 отдельно показывает применение.
