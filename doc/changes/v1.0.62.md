# v1.0.62

Date: 2026-07-20

- Updated: `wenay-common2` from `^1.0.89` to `^1.0.90`.
- Added: QA card 51 with real `Peer.createPeerPacketOffers` and
  `Peer.createPeerPacketMesh` registries, reusable in-process sessions,
  multi-hop delivery, group routing, route scoring and live fallback when the
  preferred capability disappears.
- Expanded: Media video QA card 38 now switches between a balanced 12 fps
  profile and the new unpaced MAX policy (`fps: 0`), preserving source
  resolution and reporting actual capture fps, draw rate, dropped frames and
  wire throughput.
- Kept: Wenay Calls uses a balanced publishing default and no hard-coded
  participant limit. MAX mode is an explicit diagnostics/publishing choice;
  production capacity remains a deployment/relay/SFU concern.
- Adopted transitively: common2 RPC Listen `{current: true}` wire forwarding,
  deduplicated current delivery for late subscribers and safe synchronous
  one-shot cleanup. No React wrapper change was required.
- Fixed: Jest now resolves from the project root, discovers only `*.test.*`,
  transforms TypeScript with Babel and installs Node's Encoding API in jsdom
  before common2 artifact modules load. The test stack is aligned on Jest 29.7
  instead of the incompatible Windows/pnpm Jest 30.3 runner.
- Verification: package TypeScript build, 32 Jest suites / 118 tests, and a live
  browser scenario covering best-path convergence, packet and group delivery,
  fallback routing, restoration and MAX-profile selection with no console
  errors.
