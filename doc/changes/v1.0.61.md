# v1.0.61

Date: 2026-07-20

- Added: public functional `Sparkline` component for one or more compact numeric
  series, optional per-series gradient fill, hidden series, and theme defaults.
- Added: `ResizeObserver` sizing, CSS/physical canvas separation with
  `devicePixelRatio`, finite handling for empty, zero, one-point, and constant
  data, and synchronous change-driven drawing without axes, interactions, or a
  persistent RAF loop.
- Added: a public-root export, tests for 150x23 containment, DPR resize, 30-chart
  RAF behavior, unmount cleanup, and finite coordinates, plus a visual QA card
  containing several sparklines in 26 px rows.
- Documented: migration from a local `CMiniGraph` to `Sparkline`.
- Compatibility: existing `createChartCanvas`, chart engine, and demo
  `MyChartEngine` public interfaces are unchanged.
- Updated: `wenay-common2` from `^1.0.87` to `^1.0.89`.
- Added: `useContractSlot(runtime, slotId)`, a read-only React projection of a
  common2 Contract runtime slot with status, active binding, latest binding
  event, explanations/history and explicit typed lease acquisition.
- Added: QA card 50 for v1 â†’ preferred v2 replacement, revoke/fallback,
  restore and leased calls. Contract discovery, loading, policy, Store/replay
  continuity and runtime ownership remain outside React.
- Assessed: common2 `createStoreReplicaSet` is transport/application
  infrastructure and does not need a React protocol wrapper.
- Added: a separate `#video-calls` QA rubric with a responsive product-style
  lobby and active meeting surface based on the supplied design reference.
- Added: real in-process ring/accept/hangup, live camera/microphone relay gating,
  speaker focus, participant strip, chat/people panels, skippable poll and an
  inactivity-hiding control dock in the video-call stand.
- Preserved: call and media state still comes from `usePeerCalls`,
  `useMediaSource`, `Peer.createMediaRelay` and `Media.attachVideoCanvas`; the
  new surface does not introduce a second call protocol.
- Added: public communication UI module `VideoCall` plus
  `useVideoCallController`, exported from the package root and the independent
  `wenay-react2/communication` entry. Applications inject phase, participants,
  media state, canvas ownership and call actions.
- Refactored: the QA showcase is now the canonical Peer/Media adapter for that
  public module instead of containing a private copy of the product markup.
- Expanded: the communication surface now includes browser-native screen share,
  canvas/screen WebM recording, side rooms with main-audio isolation and quiet
  return calls, moderator voice permissions, three layouts and four focus modes.
- Added: translated-caption controls plus a local voice/typed assistant command
  contract. The stand uses Web Speech when available and a deterministic RU â†’ EN
  fixture; production speech/AI translation stays application-owned.
- Replaced: `#video-calls` is no longer a QA-only mock. It is a standalone dark
  Wenay Calls surface where a named host creates a server-owned room and invite
  URL, and named guests join that URL from another browser session.
- Added: dedicated Socket.IO/RPC room authority with shared participant roster,
  chat, host voice moderation, close propagation, reconnect grace, membership-
  gated camera/microphone/screen relay and downloadable local recording.
- Removed: the provisional 12-person UI/server cap. Capacity is intentionally a
  deployment bandwidth/relay/SFU concern rather than a hard-coded product claim.
- Verification: TypeScript/package build plus a two-tab live flow covering host
  creation, invite URL, named guest join, shared roster/chat, host moderation and
  immediate meeting-close propagation.
