# v1.0.60

Date: 2026-07-16

- Updated: `wenay-common2` from `^1.0.79` to `^1.0.87`.
- Reason: adopt its bounded dynamic Store memory handling, live Media source
  state, direct-route/media-relay fixes, and additive follower/failover and
  Artifact-mirror primitives while keeping the existing React-facing APIs
  unchanged.
- Assessed: the new Artifact and Conversation clients are useful React adoption
  candidates. Concrete safe bindings, renderer boundaries, QA scenarios, and
  ownership constraints are recorded in `doc/target/my.md`; room-directory demo
  logic remains application-owned rather than becoming a speculative hook.
- Assessed: common2 1.0.84–1.0.85 changes are demo-only, 1.0.86 additions stay at
  the server/application boundary, and 1.0.87 has no runtime or public API
  changes; no existing React stand needs a new protocol or compatibility shim.
- Added: Grid Chrome now makes a context-clicked AG Grid RowNode the exclusive
  selection before copy, so right-clicking another row never accumulates an
  unintended multi-selection.
- Added: optional `commandGroups` lets applications label, order and collapse
  growing command groups without a second command or column-state API. Groups are
  keyboard-accessible, the popover scrolls within the viewport, and closed
  one-shot commands leave a short status toast.
- Improved: QA card 47 demonstrates four selectable menu directions, including
  the neutral «Панель» default, and the actual expandable columns/app groups.
- Verification: TypeScript, unit tests, a fresh package build, and the single
  local Vite stand.
